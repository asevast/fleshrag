# Техническое задание: Multimodal RAG --- семантический поиск по файлам на домашнем PC

**Версия:** 2.2 (улучшения надежности, UX и архитектуры)\
**Дата:** 2026-04-23

------------------------------------------------------------------------

## Изменения версии 2.2 (по сравнению с 2.1)

### 1. Надежность

#### 1.1 Circuit Breaker для Model Router

-   Не переключать cloud → local при первой ошибке (429/503)
-   Ввести:
    -   threshold ошибок (например, 3 подряд)
    -   cooldown (например, 60 сек)
    -   health-check перед возвратом на cloud
-   Хранить состояние провайдера в runtime state

#### 1.2 Идемпотентная индексация

-   Ввести fingerprint файла:
    -   path + size + mtime + content_hash
-   Chunk ID = hash(fingerprint + chunk_index)
-   При повторной индексации:
    -   обновлять существующие записи
    -   не создавать дубли

#### 1.3 Кэш артефактов парсинга

-   Кэшировать:
    -   OCR результат
    -   транскрипцию аудио/видео
    -   extracted frames (опционально)
-   Ключ: hash(file_content + parser_version)
-   Хранение: локально (SQLite + файловый cache)

#### 1.4 SQLite устойчивость

-   Включить:
    -   PRAGMA journal_mode=WAL;
    -   PRAGMA busy_timeout=5000;
-   Обязательно для concurrent доступа (backend + worker)

------------------------------------------------------------------------

### 2. Архитектура

#### 2.1 Версионирование индекса

-   Добавить поля:
    -   embed_model
    -   vector_dim
    -   index_version
-   При несовпадении:
    -   блокировать поиск
    -   требовать reindex

#### 2.2 Разделение settings и runtime state

-   SQLite:
    -   system_settings → конфигурация
-   Runtime state (in-memory / Redis):
    -   active_provider (с учётом fallback)
    -   circuit breaker состояние
    -   текущие ошибки

#### 2.3 Retry / Timeout политики

Разные политики: - embeddings: retry 2×, short timeout - chat: retry 1×,
longer timeout - rerank: retry 2× - transcription: без retry (дорогая
операция)

------------------------------------------------------------------------

### 3. UX/UI

#### 3.1 Отображение результатов

-   1 основной сниппет на документ
-   "Показать ещё" → дополнительные чанки
-   Показывать:
    -   путь
    -   тип файла
    -   дату изменения

#### 3.2 Явные состояния системы

UI должен показывать: - indexing in progress - using local fallback -
cloud unavailable - reindex required - budget usage (80% / 100%)

#### 3.3 Защита от опасных действий

Подтверждение (modal) для: - Clear index - Reindex all - Restart all -
Смена embedding модели

------------------------------------------------------------------------

### 4. GPU (CUDA) использование

#### 4.1 Приоритеты использования GPU (GTX 1050 Ti 4GB)

Использовать CUDA для: - faster-whisper (транскрипция) - local reranker
(cross-encoder) - embeddings (если помещается)

Не использовать GPU для: - LLM inference (нестабильно на 4GB VRAM)

#### 4.2 Auto-detect

-   Проверка CUDA при старте
-   fallback на CPU при отсутствии
-   настройка в Admin UI:
    -   "Use GPU for transcription/rerank"

------------------------------------------------------------------------

### 5. Что НЕ меняется

-   Общая архитектура (FastAPI + Celery + Qdrant)
-   Model Router концепция
-   Docker Compose структура

------------------------------------------------------------------------

## Итог

Изменения направлены на: - стабильность (circuit breaker, idempotency) -
предсказуемость (версионирование индекса) - UX (явные состояния
системы) - производительность (точечное использование GPU)

Без радикальной переработки системы.
